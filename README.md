# T 账本

一个轻量级做 T（高抛低吸）记账工具 —— 手机里随手记买入/卖出/减仓/做 T，自动算出当前平均成本、累计收益、胜率、盈亏点位。**纯前端 PWA**，零后端依赖，装到桌面像 App 一样用。

**线上地址**：<https://woshiyigebing280.github.io/t-ledger/>

---

## ✨ 核心功能

### 持仓与记账
- **多市场支持**：A 股（600519、000001…）、港股（00700、09988…）、美股（AAPL、SPCX…），填代码自动判定市场
- **三种记账入口**：记录 T 操作（高抛低吸，数量不变）/ 加仓 / 卖出减仓
- **成本结转**：移动加权平均口径，卖出部分按当前均价结转，剩余均价不变
- **删除任意一笔后**整体重算（recomputeStock），状态永远自洽
- **多币种收益 CNY 快照**：港美股盈亏按当时汇率换算成 ¥ 并固化，跨天 / 跨设备一致
- **卡片双值显示**：港美股同时显示 ¥ 收益和原币值，例如 `+¥276.45 ≈ +HK$318.00`
- **编辑持仓**：只能改代码 / 名称，或者整只删除；股数和成本价只能通过 T/加仓/减仓三个入口调整

### 趋势分析
4 张 ECharts 图：
1. **累计 T 收益**（按日累加）
2. **成本价走势**（含加仓、T、卖出标记点）
3. **胜率滚动趋势**（近 10 笔）
4. **买卖点位 + 行情**：背景叠加日 K 线，买卖点用 **B**（橙）/ **S**（绿）字母标注，按当前均价画一条参考虚线。支持 **全部 / 近 1 月 / 近 1 周** 三档筛选。

### 数据管理
- 一键导入 / 导出 JSON 备份
- 老版本 JSON 自动迁移（自动补汇率快照与 `profitCNY`）
- 完成"刷新汇率快照"后，跨设备 / 离线场景不再依赖网络
- 不会自动迁移时手动点「管理 → 刷新汇率快照」

## 🛠 技术栈

- 纯 HTML / CSS / 原生 JS，无构建工具
- [ECharts 5.4.3](https://echarts.apache.org/) 负责所有图表
- [PWA + Service Worker](./sw.js) 网络优先 + 本地缓存兜底，可"添加到主屏幕"作离线 App 用
- 数据本地存 `localStorage`，**永远不离开你的设备**

## ⚠️ 已知限制

- **美股无 K 线背景**：腾讯对部分新上市美股（如 SPCX/2026-06-12 上市的 SpaceX）只返回 1 条，雅虎有数据但无 CORS。当前选择"只画 B/S 点"。
- **港美股交易新建时需要联网**（assignProfitCNY 触发）拉一次汇率；离线时使用兜底汇率 USD=7.2 / HKD=0.92
- **首次打开新版（v19+）**若已有港美股数据，需要联网一次完成迁移；或手动点「管理 → 刷新汇率快照」
- 港美股腾讯接口不返回前复权数据，若该股历史上发生过拆股/送转，K 线背景可能有视觉跳空
- 软删除单笔记录后整体重算（recomputeStock），状态保持自洽，但**编辑历史 T 时汇率按最新重算**——这意味着你今天编辑几个月前的港美股 T，`profitCNY` 会按今天的汇率覆盖原快照
